# Pre-v0.5 Hardening Audit — 2026-05-05

Six parallel read-only audits run across CLI, TUI, Dashboard, Core modules,
Code health, and Test suite. Total raw findings: ~300. This doc is the
prioritized punch-list. Tackle CRITICAL before v0.5 work starts; the rest
forms a hardening track that can run alongside feature work.

Surface | Findings (raw) | Top 3
---|---|---
CLI | 80 | `urika logs` doesn't show logs · `urika tui` launches a different binary than `urika` · `urika new --json` silently overwrites projects
TUI | ~22 | `/setup` registered but no handler · `hide_input=True` prompts hang under bridge · 5 CLI features have no slash command
Dashboard | ~70 | `project_settings.html` missing v0.4.1 fields · CDN-only deps break offline · 2 pages reachable only via deeplink
Core | 80 | non-atomic writes everywhere (vault, registry, sessions, advisor memory) · `loop.py:496` uses stale `run` for leaderboard · privacy parse-failure treated as "open"

## CRITICAL — fix before starting v0.5

These are correctness, security, or privacy issues. Several are user-visible
data-loss or contract-violation paths.

### C1 — Atomic-write the JSON state files. **Severity: high.**
Five sites write JSON state with `path.write_text(...)` directly. A SIGTERM
mid-write produces corrupt JSON. The file lock (where present) only
serializes processes that obey it; the write itself is still non-atomic.

| Site | Effect on corruption |
|---|---|
| `core/vault.py:104, 131, 244` | secrets file truncated → user has to re-enter all keys |
| `core/registry.py:40` | master `~/.urika/projects.json` gone → `urika list` empty |
| `core/orchestrator_sessions.py:51` + `core/session.py:39` | session lost; `list_sessions` silently skips broken file |
| `core/advisor_memory.py:57` | conversation history silently reset to `[]` (line 43-45 swallows the JSONDecodeError) |
| `core/progress.py:37` + `core/criteria.py:111` + `core/method_registry.py:34` + `core/usage.py:67` | progress/criteria/usage silently truncated under crash |

Fix: one helper `core/atomic_write.py::write_json_atomic(path, data)` using
`os.open(O_CREAT|O_EXCL, 0o600) → write → os.replace(tmp, path)`. Adopt at
all 9 sites. ~2 hours including tests.

### C2 — Privacy parse-failure treated as "open mode, allow." **Severity: high.**
`core/privacy.py:27` and `core/agents/config.py:347` both `except Exception:
return open_mode_default` when `urika.toml` fails to parse. A corrupt or
malformed config file silently bypasses the privacy guard and may route a
private project's traffic to cloud.

Fix: distinguish `FileNotFoundError` (legitimate "no project config")
from `tomllib.TOMLDecodeError` (parse failure → fail loud). 30 min.

### C3 — `loop.py:496` leaderboard uses stale `run`. **Severity: high.**
Leaderboard update is outside the inner `for run in runs:` loop, so it
captures only the last run's metrics for *all* runs. Method ranking
silently incorrect. Fix: move leaderboard call into the inner loop. 15 min.

### C4 — `urika new --json` silently overwrites existing projects. **Severity: high (data loss).**
`cli/project_new.py:343-348` calls `shutil.rmtree` on JSON-mode without
confirmation or `--overwrite` flag. A user scripting `urika new` over an
existing project name loses their work.

Fix: require `--overwrite` flag in JSON mode. 20 min.

### C5 — `urika tui` launches the TS binary while `urika` (no args) launches the Python TUI. **Severity: high (UX).**
Two completely different "TUIs" reachable by similar commands. Source of
support tickets waiting to happen.

Fix: rename the TS-launching command to `urika tui-ts` or
`urika experimental-tui`, OR remove the TS launcher (the Python Textual TUI
is the default and is feature-complete). Document the choice in the
release notes. 1 hour.

### C6 — `urika logs` doesn't show logs. **Severity: high.**
`cli/data.py:152-194` docstring says "Show experiment run log" but only
prints progress.json + status. Doesn't open `experiments/<id>/run.log`.

Fix: tail the actual log file (the dashboard already does this). 30 min.

### C7 — `/setup` declared as worker but never registered. **Severity: high (new user blocker).**
`tui/app.py:34` lists `"setup"` in `_WORKER_COMMANDS`, but no
`@command("setup")` exists in `repl/commands*.py`. Dispatch falls through
to "Unknown command: /setup". A new TUI user cannot run setup without
dropping to a shell.

Fix: register `cmd_setup` mirroring `cmd_config`. 15 min.

### C8 — `hide_input=True` prompts silently hang under TUI bridge. **Severity: high.**
`click.prompt(hide_input=True)` (used in `cli/config.py:61` and
`config_notifications.py:457` for API key + password entry) calls
`getpass.getpass`, which on POSIX opens `/dev/tty` directly and bypasses
both `sys.stdin` and the bridge's pipe. **API key entry from inside the
TUI freezes.**

Fix: monkeypatch `click.termui.hidden_prompt_func` to a fallback that uses
`input()` when the TUI bridge is active. ~45 min including a test.

### C9 — `project_settings.html` missing v0.4.1 endpoint fields. **Severity: high.**
The Privacy tab on the per-project settings page does not have the
`context_window` / `max_output_tokens` form rows that
`global_settings.html:248-269` gained in v0.4.1. Per-project endpoint
overrides cannot set these without round-tripping through the global tab —
silently ignoring user intent.

Fix: copy the form rows into project_settings.html. Wire the existing
`/api/projects/{name}/settings` parser to accept them (it was extended in
v0.4.1; verify the parser regex covers both scopes). 30 min.

### C10 — `core/orchestrator_sessions.py` ID collision on rapid calls. **Severity: med→high.**
`_timestamp_id` returns second-resolution ISO timestamp. Two saves within
the same second collide; the second overwrites the first AND prune sorts
them identically.

Fix: add a millisecond suffix or counter. 15 min.

## HIGH — week-1 hardening (~10–15 dev-hours)

### H1 — Dashboard CDN-only deps break offline use.
htmx, Alpine, Chart.js currently load from CDN. A user without internet
cannot use the dashboard. Vendor them under
`src/urika/dashboard/static/vendor/` and bundle via hatch
force-include. ~1 hour.

### H2 — `/projects/{name}/compare` and `/criteria` exist but no nav reaches them.
`experiment_compare.html` and the criteria page render correctly but the
sidebar has no link. Either add nav or document as deeplink-only.
30 min.

### H3 — SSE streams lack disconnect detection.
Dashboard `/api/runs/{id}/stream`, `/api/finalize/stream`,
`/api/advisor/stream` keep the spawn process alive after the client
disconnects (browser tab closed, network drop). Resource leak under heavy
use. Add `request.is_disconnected()` polling. ~1 hour.

### H4 — `core/updates.py` version comparison wrong for pre-releases.
`_parse_version` truncates at first non-int. `0.4.0rc1` parses to
`(0,4,0)` and is treated equal to `0.4.0`. Replace with
`packaging.version.Version`. 20 min.

### H5 — `agents/config.py:194` — privacy URL match too permissive.
`"anthropic.com" in url` matches `http://anthropic.com.evil.com`. Use
`urlparse(url).hostname.endswith("anthropic.com")`. 10 min.

### H6 — `claude_sdk.py` token aggregation conflates fresh vs cached.
`tokens_in = input + cache_create + cache_read` is shown to dashboard
Usage tab as "real" usage; overstates cost reasoning. Track the three
separately and derive the aggregate at display time. 45 min.

### H7 — `email_channel.py:67` — passwordless SMTP false-positive.
`health_check` returns `(True, "")` for relays that forbid anonymous send.
Add a STARTTLS noop or auth check. 30 min.

### H8 — TUI: 5 CLI features have no slash command.
`/summarize`, `/sessions list`, `/memory list/show/add/delete`,
`/venv create/status`, `/experiment-create`. Memory is the v0.4 Track 2
flagship and the most painful gap. ~2 hours to add stubs.

### H9 — TUI: subprocess stdin not bridged.
OrchestratorChat shells out via Bash; child inherits the bridge's empty
pipe as stdin. Any subagent that reads stdin hangs. Pass
`stdin=subprocess.DEVNULL` to those Bash invocations. 20 min.

### H10 — `cli/run.py:1098` lines, single `run()` function ~900 lines.
Extract `_run_single(...)` and `_run_meta(...)` private helpers; the body
duplicates ~100 lines between the two paths
(`cli/run.py:430-528` vs `818-934`). Refactor target — not a bug, but
contributors hit it constantly. ~3 hours.

### H11 — `progress.py` "best run" picker doesn't honor lower-is-better.
`cli/run_planning.py:151-156` `_best_metric_val` picks `max()` regardless.
Same bug was fixed in `cli/run.py:970-1000` for notification summaries
but not propagated. Reuse `_LOWER_IS_BETTER` from labbook. 20 min.

### H12 — `tests/test_cli_memory.py` doesn't exist.
`urika memory` is v0.4 Track 2's flagship feature. Untested.
Add `tests/test_cli_memory.py` with `CliRunner` covering list/show/add/delete.
~45 min.

### H13 — `time.sleep(1.1)×7` in orchestrator session tests.
`tests/test_orchestrator_sessions.py` accepts ~9 seconds of wall-clock
just to keep timestamp ordering distinct. Combined with C10 (millisecond
suffix), monkeypatch the timestamp generator and drop the sleeps.
30 min.

### H14 — Vault file backend race window.
`vault.py:269-289` writes the secrets file with `write_text` then chmods
to 0o600. Window between create and chmod allows another user on a
multi-user host to read it. Use
`os.open(..., O_CREAT|O_EXCL, 0o600)`. 20 min.

### H15 — `compliance.py` scrub list may have drifted.
`_OAUTH_TOKEN_VARS` is short; newer Claude Code versions may add markers
not on the list. Verify against `strings $(which claude) | grep -i
claude_code_` and bring the scrub list up to date. 15 min.

### M13 — `cli/_base.py:62-87` UserCancelled detected by string compare.
`type(exc).__name__ == "UserCancelled"` instead of importing the class.

### M14 — Replace SSE-stream `time.sleep(0.5)` in dashboard tests with `wait_until`.
`tests/test_dashboard/test_api_stream.py`, `test_api_finalize.py`,
`test_runs_helpers.py`. ~45 min.

### M15 — Register `slow` and `integration` pytest markers.
`pyproject.toml [tool.pytest.ini_options]` is bare. Add markers,
mark heavy files, document `pytest -m "not slow"` for default fast loop.
30 min.

### M16 — Promote `dev/scripts/smoke-v04-e2e-open.sh` to a `@pytest.mark.integration` test.
Hybrid-privacy E2E currently runs only as a shell script. Wrap it as an
integration-marker pytest. 1 hour.

### M17 — Add known-answer numerical asserts to thin tool tests.

Add r²/accuracy on a controlled fixture. ~90 min.

### M18 — `cli_display.py:69` `_IS_TTY` evaluated once at import.
TUI-time stdout swap leaves it stale; spinners become no-ops in the TUI
when the panel could render. Re-check at use, or expose a setter.
20 min.

### M19 — Vault non-empty-string secrets reported as unset.
`vault.py:455` `gval if gval else None` — empty string secret reported
as missing. Niche but legitimate. 10 min.

### M20 — Replace `pytest.skip("$path 404")` in `test_no_api_links.py` with route-aware fixtures.

`xfail(strict=True)` if the 404 is intentional. 30 min.
