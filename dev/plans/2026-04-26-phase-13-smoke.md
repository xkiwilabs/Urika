## Manual browser checklist — pending

These are user-facing checks pytest cannot exercise. Walk through them
against a `urika dashboard` running on a project that has at least one
completed experiment, a finalized findings file, registered data
sources, and recorded usage sessions.

### Phase 13A — Modal flag expansion

- [ ] **+ New experiment** modal on `/projects/<n>/experiments`: form shows
      Instructions, Max turns, Audience, Auto checkbox, Max experiments,
      Review criteria, Resume. Submitting forwards every set field to
      `urika run` (verify in `experiments/<id>/run.log` first lines).
- [ ] **Finalize project** modal on project home: Instructions, Audience,
      Draft checkbox. Setting Draft = on writes to `projectbook/draft/`,
      not over `projectbook/report.md`.
- [ ] **+ New project** modal on `/projects`: Instructions field present;
      builder agent receives it (verify in builder output).

### Phase 13B — Per-experiment agent buttons

- [ ] On `/projects/<n>/experiments/<id>`, when no `report.md` exists,
      "Generate report" button shows. After running, label flips to
      "Re-generate report" + a "View report" link.
- [ ] Same flip behaviour for "Generate / Re-generate presentation".
- [ ] **Evaluate** button always shown; clicking spawns evaluator and
      redirects to live log.
- [ ] All three modals show their flag fields (instructions textarea,
      audience picker for report + presentation).

### Phase 13C — Summarize

- [ ] **Summarize project** button on `/projects/<n>`. After successful
      run, button label flips to "Re-summarize project" and a "Summary"
      card appears in Final outputs.
- [ ] Modal has Instructions textarea (optional). Setting an instruction
      shows up appended to the prompt in `projectbook/summarize.log`.
- [ ] Click the "Summary" card → renders `projectbook/summary.md` as
      formatted markdown, breadcrumb ends at "Project summary".
- [ ] Privacy gate: project in private mode without an endpoint → modal
      submission returns 422; no subprocess spawned.

### Phase 13D — Build tool

- [ ] On `/projects/<n>/tools` (project tools page), `+ Build tool`
      button visible. Modal requires instructions textarea.
- [ ] Submitting empty instructions → 422 with "instructions is
      required".
- [ ] Submitting valid instructions → spawns `urika build-tool`,
      redirects to `/projects/<n>/tools/build/log`. After completion,
      a new `.py` file appears under `<project>/tools/`.
- [ ] On the global `/tools` page, NO `+ Build tool` button.
- [ ] Privacy gate: private mode without endpoint → 422; no spawn.

### Phase 13E — Data inspection

- [ ] `/projects/<n>/data` lists registered data files with size and
      detected format.
- [ ] Clicking an entry → `/data/inspect?path=...` shows columns,
      dtypes, missing counts, numeric stats, head(10), tail(10).
- [ ] Path traversal attempt (`?path=/etc/passwd` or `?path=../../foo`)
      → 400.
- [ ] Unsupported file extension → page renders 200 with
      "Unsupported format" notice (not a 500 crash).
- [ ] Empty `data_paths` → empty state with helpful hint.

### Phase 13F — Usage page

- [ ] `/projects/<n>/usage` shows Totals card with sessions, tokens,
      cost, agent calls, experiments.
- [ ] Two line charts render: tokens-over-time and cost-over-time.
      Hovering a point shows tooltip with the value.
- [ ] Chart axis colours flip when toggling theme (light ↔ dark).
- [ ] Recent sessions table renders newest first, capped at 50 rows.
- [ ] Project with no `usage.json` → empty state.

### Phase 13G.1 — `action_label` macro

- [ ] No regressions on label flips: confirm "Generate / Re-generate"
      flips correctly on report, presentation, finalize, summarize when
      the corresponding artifact appears or disappears on disk.

### Phase 13G.2 — Sidebar order

- [ ] Sidebar order inside a project: Home, Experiments, Methods,
      Tools, **Data**, Knowledge, Advisor, **Usage**, Settings.
- [ ] Each link's active state lights up when navigating to it
