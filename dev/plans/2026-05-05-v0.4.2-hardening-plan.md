## Refuted items — drop from plan

- **M7 — Delete `repl_session.py`.** Refuted. Still imported by 6 test

  flow OR delete tests + function together. Drops off this plan.
- **M17 — Thin tool tests have shape-only asserts.** Refuted. All 6 cited
  tests have at least one numerical threshold (e.g. `r2 > 0.5`, `r2 >=

  hardening track can still tighten thresholds opportunistically.

## Partial items — reframed

- **C2 — Privacy parse-failure.** Confirmed in `core/privacy.py:24-28`
  (bare `except Exception: return True`). The `agents/config.py:347-352`
  side is narrower (catches `OSError, ValueError, KeyError, TypeError`
  and *logs* a warning) — but `tomllib.TOMLDecodeError` is a `ValueError`
  subclass, so it IS caught and the function still falls back to
  `RuntimeConfig()` with `privacy_mode="open"`. **Same effect, just
  louder.** Both sites must be fixed.
- **H6 — Token aggregation conflates fresh vs cached.** Confirmed for
  user-visible surfaces (cost estimation, dashboard usage tab,
  `AgentResult`/`RunRecord`). The broken-out fields ARE tracked
  internally and emitted to the prompt-size trace — they're just not
  propagated upward. Fix is a propagation+display change, not a tracking
  change.
- **H9 — TUI subprocess stdin not bridged.** The audit's mechanism is
  wrong — `OrchestratorChat` doesn't spawn subprocesses directly; the
  SDK does. But the *real risk* persists: when the SDK-spawned `claude`
  process invokes `urika` (e.g. via Bash tool with `allowed_bash=
  ["urika"]`), and that grandchild hits `getpass.getpass()` (the C8
  problem), the prompt also can't reach the user. Drop this as its own
  item; subsume into C8's fix scope.

## Package A — Atomicity & corruption resistance (~6 hours)

Closes: **C1 (9 sites), C10, H14, M11, M19**

The biggest single correctness win. Today, any SIGTERM or crash mid-write
on a JSON state file produces silent corruption; secrets file has a
write-then-chmod race; advisor history is silently zeroed on read
failure.

### A.1 — Add `core/atomic_write.py`

New module exposing two helpers:

```python
def write_text_atomic(path: Path, body: str, *, mode: int = 0o644) -> None:
    """Write body to path atomically, then fsync directory."""
    # tmp = path.with_suffix(path.suffix + f".{os.getpid()}.tmp")
    # fd = os.open(tmp, os.O_WRONLY | os.O_CREAT | os.O_EXCL, mode)
    # try: os.write(fd, body.encode("utf-8"))
    # finally: os.close(fd)
    # os.replace(tmp, path)
    # fsync parent dir for crash-safety on power loss

def write_json_atomic(path: Path, data: Any, *, mode: int = 0o644) -> None:
    write_text_atomic(path, json.dumps(data, indent=2) + "\n", mode=mode)
```

`mode=0o600` overrides for secret files (closes H14 — file is created
with restrictive perms via `O_CREAT|O_EXCL` instead of write-then-chmod).

### A.2 — Migrate the 9 write sites

from urika.core.labbook import _LOWER_IS_BETTER

def _best_metric_val(m: dict) -> float:
    metrics = m.get("metrics", {})
    nums = {k: v for k, v in metrics.items() if isinstance(v, (int, float))}
    if not nums:
        return float("-inf")
    # Prefer first non-lower-is-better metric for ranking
    primary = next((k for k in nums if k not in _LOWER_IS_BETTER), None)
    if primary:

    # All lower-is-better — invert
    return -min(nums.values())

### E.2 — Bridge `getpass.getpass` (C8)

Add to `src/urika/tui/agent_worker.py` (or wherever the worker bootstraps
the bridge):

```python
def _patch_getpass_for_bridge():
    import click.termui
    _original = click.termui.hidden_prompt_func

    def _bridged(prompt: str) -> str:
        if hasattr(sys.stdin, "_tui_bridge"):  # Set by _TuiStdinReader
            click.echo(prompt + " (input hidden)", nl=False)
            return sys.stdin.readline().rstrip("\n")
        return _original(prompt)

    click.termui.hidden_prompt_func = _bridged
```

Mark the `_TuiStdinReader` instance with a sentinel attribute. Do NOT
patch `getpass.getpass` itself — many libraries import it as a function
reference and patches won't take effect.

This single fix closes the audit's H9 partial as well — when SDK-Bash
spawns `urika`, those grandchildren run in their own process and don't
hit the TUI bridge directly; they hit the parent terminal. The fix
ensures the *parent* TUI handles its own getpass.

Hits 3 sites in `cli/config.py` (lines 59, 803, 1008) and 1 in
`cli/config_notifications.py:454-459` automatically.
